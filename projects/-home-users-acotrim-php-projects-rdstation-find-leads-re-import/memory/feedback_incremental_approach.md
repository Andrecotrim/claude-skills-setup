---
name: Abordagem incremental no projeto
description: Usuário prefere construir o projeto passo a passo, sem criar tudo de uma vez
type: feedback
---

Não criar toda a estrutura do projeto de uma vez. Trazer funcionalidades aos poucos para evitar código desnecessário.

**Why:** O usuário quer controle sobre o que entra no projeto e evitar desperdício.

**How to apply:** Sempre perguntar qual é a próxima parte antes de implementar. Implementar uma camada/módulo por vez e aguardar feedback antes de continuar.
