Nunca executar `git commit` sem que o usuário autorize explicitamente.

**Why:** O usuário quer controle total sobre o histórico do repositório. Commitar por conta própria ao final de uma tarefa não é bem-vindo.

**How to apply:** Após implementar qualquer feature ou correção, parar antes do commit e apresentar o que foi feito. Só commitar quando o usuário disser algo como "pode commitar", "faz o commit" ou similar.
